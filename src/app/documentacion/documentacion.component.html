Documentación del proyecto

Estructura de componentes y plantillas

El proyecto se organiza en varios componentes principales:

  • AppComponent: Componente principal de la aplicación, que contiene el enrutamiento y el componente MenuComponent.
  • MenuComponent: Componente que muestra el menú de navegación y controla la vista actual.
  • CalendarioComponent: Componente que muestra el calendario y maneja las operaciones relacionadas con eventos.
  • AgendaComponent: Componente que muestra la agenda y maneja las operaciones relacionadas con eventos.
  • DocumentacionComponent: Componente que muestra la documentación de la aplicación.

El archivo app.component.html contiene el enrutador y el componente MenuComponent. A través del enrutador, se cargará el componente correspondiente según la vista seleccionada en el menú.

La estructura de las plantillas de cada componente se describe detalladamente en los archivos .html correspondientes.

Estructura del código e invocación de funciones

En el archivo menu.component.ts, el componente MenuComponent tiene una propiedad vista que controla la vista actual. Se utiliza en la plantilla (menu.component.html) para mostrar el contenido correspondiente según la vista seleccionada.

El componente MenuComponent también define métodos para cambiar la vista, como cambiarVista(vista: string), que actualiza la propiedad vista con el valor seleccionado.

El componente CalendarioComponent y AgendaComponent también tienen métodos para realizar operaciones relacionadas con eventos, como agregar eventos, ver detalles de eventos, marcar eventos como finalizados, etc. Estos métodos se invocan desde las plantillas correspondientes.

Estructuras de datos utilizadas

El proyecto utiliza las siguientes estructuras de datos:

  • Evento: una interfaz o clase que representa un evento, con propiedades como título, descripción, fecha, hora de inicio, hora de fin, etc.
  • Dia: una interfaz o clase que representa un día en el calendario o la agenda, con propiedades como número de día, estado (hoy, otro mes, evento, seleccionado, etc.), y una lista de eventos para ese día.

Funciones del componente Calendario

  • cambiarPestana(): Esta función se utiliza para cambiar la pestaña activa en el Calendario. Recibe como parámetro la pestaña seleccionada y actualiza la propiedad pestanaActual con el valor correspondiente.
  • agregarEvento(): Esta función se ejecuta cuando se hace clic en un día del Calendario para agregar un evento. Solicita al usuario información como título, descripción, hora de inicio, hora de fin y estado de finalización del evento. Luego, crea un objeto de evento con la información ingresada y lo agrega a la lista de eventos del día y a la lista global de eventos.
  • verEvento(evento): Esta función se activa al hacer clic en un evento del Calendario. Muestra los detalles del evento, como título, descripción, hora de inicio, hora de fin y estado de finalización. También ofrece la opción de cambiar el estado de finalización del evento.
  • borrarEvento(): Esta función se utiliza para borrar un evento del Calendario. Recibe como parámetros el día y el evento específico a borrar. Elimina el evento de la lista de eventos del día y de la lista global de eventos.
  • mesAnterior(): Esta función se ejecuta al hacer clic en el botón para ir al mes anterior. Disminuye el mes actual en 1 y genera el nuevo calendario.
  • mesSiguiente(): Esta función se activa al hacer clic en el botón para ir al mes siguiente. Aumenta el mes actual en 1 y genera el nuevo calendario.
  • resetearCalendario(): Esta función se utiliza para restablecer el Calendario. Limpia la lista de semanas y genera un nuevo calendario.

Funciones del componente Agenda

  • cambiarEstadoEvento(evento: any): Esta función se ejecuta cuando se hace clic en el botón para cambiar el estado de finalización de un evento en la Agenda. Recibe como parámetro el evento específico y cambia su estado de finalización (de finalizado a no finalizado o viceversa).
  • editarEvento(evento: any): Esta función se activa al hacer clic en el botón de editar evento en la Agenda. Permite al usuario editar el título, descripción, hora de inicio, hora de fin y estado de finalización del evento. Guarda los cambios realizados en el almacenamiento local y actualiza la lista de eventos.

Problemas encontrados y soluciones

A lo largo del desarrollo del proyecto, es posible que se hayan encontrado algunos problemas y desafíos. A continuación se describen algunos de ellos junto con las soluciones correspondientes:

  • Problema: Error al cargar la vista de documentación en el componente MenuComponent debido a un error de tipo.
  • Solución: El error se resolvió asegurándose de que el tipo de datos utilizado en la propiedad vista coincidiera con las opciones del menú.

Posibles mejoras

A continuación se presentan algunas posibles mejoras para el proyecto:

  • Implementar autenticación y gestión de usuarios para permitir que cada usuario tenga su propio calendario y agenda personalizados.
  • Mejorar la usabilidad y la interfaz de usuario agregando funcionalidades como arrastrar y soltar eventos, notificaciones, etc.
  • Agregar funcionalidades adicionales, como recordatorios, compartir eventos con otros usuarios, búsqueda y filtrado avanzados, etc.
